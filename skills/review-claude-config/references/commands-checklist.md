# Commands (Slash Commands) Review Checklist

## Frontmatter
- [ ] Has `description` field explaining what the command does

## Naming
- [ ] Command name is clear and matches user intent
- [ ] Uses kebab-case convention
- [ ] Not too long (ideally 1-3 words)

## Content
- [ ] Instructions are specific about what to do when invoked
- [ ] Delegates to appropriate agents/skills when the task is complex
- [ ] Not duplicating logic that belongs in a skill or agent

## Discoverability
- [ ] Command name is intuitive — user can guess it
- [ ] Description in frontmatter helps Claude match user intent
